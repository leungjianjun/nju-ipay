	SVN使用规则
--------------------------------------------------------
1.第一次使用请checkout，每次每天开始编码前要先update，完成
一部分功能后要commit
2.每个包尽可能由一个人负责，不要尝试两个人同时编写同一个文件，
这冲突很难处理，要修改别人文件时先和负责这个包的人负责，全局
文件修改，建包，修改目录结构和TM说一下
3.强烈建议安装subclipse插件。
========================================================
	文件目录结构
--------------------------------------------------------
docs：存放文档
source：存放源码
	server：存放服务器源码
	client：存放客户端源码
========================================================
	构建服务器代码
--------------------------------------------------------
1.服务器使用Maven构建项目，同样强烈推荐客户端也使用maven构建
2.服务器推荐IDE是STS（spring tool suite）+subclipse或者自行在
jee eclipse安装m2eclipse和subclipse等插件
3.服务器文件目录完全按照Maven的webapp文件目录
	src/main/java ：源码
	src/main/resources ：源码资源
	src/test/java ：测试代码
	src/test/resources ：测试资源
	src/main/webapp/
			resources ：网站的资源，js，css等
			WEB_INF
				classes
				spring ：spring配置文件
				views ：html、jsp文件类型的
				web.xml ：服务器配置文件
---------------------------------------------------------
	服务器框架
---------------------------------------------------------
1.服务器作用分两部分：一是处理客户端请求，二是建立一个后台管理页面
2.服务器已spring 3 MVC框架为主体
3.数据处理用hibernate框架处理
